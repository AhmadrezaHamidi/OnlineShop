💳 ۴. Flow کامل پرداخت‌ها
💳 پرداخت زرین‌پال (آنلاین)
۱
مشتری سفارش ثبت می‌کند
POST /api/v1/orders { paymentMethod:"ZarinPal" }
۲
آیتم‌ها اضافه می‌شوند
POST /orders/{id}/items (موجودی Reserve می‌شود)
۳
درخواست پرداخت
POST /orders/{id}/payments → Authority
۴
Redirect به درگاه زرین‌پال
zarinpal.com/pg/StartPay/{authority}
۵
بازگشت به Callback
GET /payment/callback?Status=OK&Authority=xxx
۶
Verify با ZarinPal API
POST zarinpal.com/pg/v4/payment/verify.json
۷
تأیید → سفارش Confirmed ✅
PATCH /orders/{id}/payments/{pid}/complete → Status:Confirmed
شکست: /payments/{pid}/fail → موجودی Release → Status:Pending
🚚 پرداخت درب منزل (COD)
۱
مشتری سفارش ثبت می‌کند
POST /api/v1/orders { paymentMethod:"CashOnDelivery" }
۲
آیتم‌ها + نهایی‌سازی
POST /items → POST /place → Status:Pending
۳
ادمین سفارش را تأیید می‌کند
PATCH /orders/{id}/confirm → Status:Confirmed
۴
ارسال می‌شود
PATCH /orders/{id}/ship → Status:Shipped
۵
تحویل + دریافت پول نقد
POST /orders/{id}/payments (COD)
PATCH /payments/{pid}/complete
۶
تأیید تحویل ✅
PATCH /orders/{id}/deliver → Status:Delivered
موجودی بعد از complete به صورت قطعی کسر می‌شود (ConfirmStock)
🛒 ۵. Flow کامل سفارش — همه مراحل
چرخه وضعیت:
⏳ Pending → ✅ Confirmed → 📦 Shipped → 🏠 Delivered | هر مرحله → ❌ Cancelled
وضعیتAPIMethodعاملتأثیر موجودی
Pending /api/v1/orders POST Customer —
+ آیتم /orders/{id}/items POST Customer Reserve ↑
- آیتم /orders/{id}/items/{itemId} DELETE Customer Release ↓
ثبت پرداخت /orders/{id}/payments POST Customer/System —
Confirmed /orders/{id}/payments/{pid}/complete PATCH System/Admin Confirm (کسر قطعی)
Shipped /orders/{id}/ship PATCH OrderManager —
Delivered /orders/{id}/deliver PATCH OrderManager —
Cancelled /orders/{id}/cancel PATCH Customer / Admin Release همه رزروها ↓
ℹ️ قوانین موجودی: AddItem → Reserve | RemoveItem → Release | Cancel → Release همه | PaymentComplete → Confirm (کسر دائمی)
